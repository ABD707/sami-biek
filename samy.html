النصائح والتحليل الذكي 🧠

تحليل تلقائي لأداء المطعم مع اقتراحات قابلة للتنفيذ.

الإعدادات

☁️ المزامنة السحابية (Firebase)

جاري التحقق...
🔒 خطوات تأمين Firebase (مهم جداً)
  1. تفعيل المصادقة المجهولة (Anonymous Auth):
    ادخل على Authentication → Sign-in method → فعّل Anonymous → Save.
  2. قواعد قاعدة البيانات الآمنة:
    من Realtime Database → Rules ضع التالي ثم Publish: 
    {
  "rules": {
    "samybek": {
      ".read":  "auth != null",
      ".write": "auth != null"
    }
  }
}
    هذه القواعد تمنع أي شخص بدون auth token من قراءة/كتابة البيانات، حتى لو عرف رابط قاعدة البيانات.
  3. أعد فتح الموقع. ستظهر علامة "السحابة متصلة" أعلى الصفحة.
📖 إنشاء مشروع Firebase من الصفر (إذا لزم)
  1. ادخل على console.firebase.google.com وأنشئ مشروع.
  2. Build → Realtime Database → Create Database.
  3. Project Settings ⚙️ → Your apps → Web → انسخ firebaseConfig والصقه في الـ HTML.
  4. طبّق خطوات الأمان أعلاه.
🛡️ طبقات الحماية المفعّلة:
  • تشفير AES-256-GCM لكل البيانات (سحابياً ومحلياً)
  • PBKDF2 بـ 250,000 دورة لاشتقاق المفتاح من كلمة المرور
  • جلسة محفوظة مشفّرة بمفتاح خاص بالجهاز (Device-bound AES-GCM)
  • قفل تلقائي بعد 5 محاولات دخول فاشلة (Brute-force lockout تصاعدي)
  • Firebase Anonymous Auth + Rules تمنع الوصول غير المُصادَق
  • Content Security Policy + تعطيل DevTools shortcuts
